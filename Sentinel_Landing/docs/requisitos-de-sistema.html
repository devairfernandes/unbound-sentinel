Requisitos de Sistema

Saiba qual é o hardware recomendado para rodar o Sentinel DNS de forma otimizada no seu provedor.

Nota: O Sentinel DNS é distribuído como uma ISO Appliance nativa. Atualmente, nossa versão de produção (Stable) roda sobre a base sólida do Rocky Linux 9. A ISO já formata o disco e instala o sistema base com o Sentinel pré-configurado e ajustado no kernel.

Early Access (Beta): Já estamos iniciando a fase de testes Beta com o nosso core portado para o novíssimo Rocky Linux 10.2. Participantes do programa de Early Access poderão homologar esta nova engine em breve.

Hardware Recomendado (Produção ISP)

Para ambientes de produção com tráfego intenso (acima de 20.000 requisições por segundo), recomendamos o seguinte hardware físico ou virtualizado:

  • Processador: 8 Cores ou mais (Intel Xeon ou AMD EPYC recomendados).
  • Memória RAM: 16 GB ou mais. O Sentinel usa agressivamente a RAM para manter o cache de DNS e listas da CTI na memória, garantindo respostas em <1ms.
  • Armazenamento: 50 GB NVMe ou SSD Enterprise. Discos rápidos são vitais para a gravação dos logs do SIEM sem gargalos de I/O.
  • Rede: Placas de rede de 10 Gbps (SFP+), preferencialmente Intel (ex: X520, X710) configuradas em LACP (Bonding).

Requisitos Mínimos (Laboratório/Homologação)

Para pequenos provedores ou testes de laboratório (abaixo de 5.000 QPS):

  • 2 vCPU
  • 4 GB RAM
  • 20 GB SSD
  • Rede 1 Gbps

Ambientes Virtuais (Hypervisors)

O Sentinel DNS roda perfeitamente de forma virtualizada, desde que as placas de rede virtuais não se tornem o gargalo. Suportamos nativamente:

  • Proxmox VE (Recomendado usar placas VirtIO)
  • VMware ESXi (Recomendado VMXNET3)
  • Bare Metal (Hardware físico dedicado - Melhor performance)
