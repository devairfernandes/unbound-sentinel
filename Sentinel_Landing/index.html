Perguntas Frequentes

O que é o Sentinel DNS?

O Sentinel DNS é um appliance open-source de DNS recursivo e firewall, baseado no motor Unbound, voltado para Provedores de Internet (ISPs) e redes corporativas que precisam bloquear ameaças na camada DNS antes que elas atinjam os usuários finais.

Como funciona o bloqueio de ameaças?

O Sentinel sincroniza listas de domínios, URLs e IPs maliciosos a partir de fontes de Cyber Threat Intelligence (CTI) e bloqueia a resolução desses endereços diretamente no DNS, impedindo que dispositivos da rede acessem servidores de phishing, malware ou botnets.

Preciso de conhecimento avançado em Linux para instalar?

Não. A instalação é guiada por um assistente de primeiro boot que detecta automaticamente hardware e interfaces de rede, dispensando edição manual de arquivos de configuração do Unbound ou do sistema operacional.

Qual a diferença entre os planos FREE, PRO Elite e PRO Lite?

O plano FREE oferece proteção DNS básica para até 1 nó satélite, sem painel corporativo. O PRO Elite inclui proteção CTI avançada, bloqueio de malware Zero-Day e cluster de satélites ilimitado, com cobrança mensal. O PRO Lite é uma licença vitalícia de pagamento único, com recursos base do PRO e dashboard básico incluído.

O Sentinel funciona com IPv6?

Sim. O Sentinel DNS e o motor Unbound subjacente têm suporte nativo a IPv6, tanto para escuta de requisições de clientes quanto para recursão em servidores raiz e autoritativos globais.

Como funciona o AnaBlock?

O AnaBlock é um módulo do Sentinel DNS focado no bloqueio automatizado de sites listados por ordem judicial da ANATEL. Ele consulta listas oficiais periodicamente e injeta regras locais (RPZ) sem exigir intervenção humana.

O que é Cyber Threat Intelligence (CTI)?

CTI é o processo de coleta e análise de dados sobre ameaças cibernéticas globais. O Sentinel usa feeds OSINT de CTI para bloquear acesso a botnets, phishings e ransomware em tempo real via DNS Firewall.

Posso integrar o Sentinel ao meu Mikrotik ou BNG?

Sim. Basta configurar o IP do Sentinel como o DNS primário e secundário entregue via DHCP, PPPoE ou estático no seu BNG (Mikrotik, Huawei, Cisco). O tráfego DNS passará a ser protegido imediatamente.

Qual o sistema operacional recomendado?

O Sentinel DNS é otimizado e empacotado nativamente como um appliance em Rocky Linux (compatível com RHEL). Oferecemos uma ISO completa para facilitar o deploy Bare Metal ou em Hypervisors (Proxmox, VMware).

Existe limite de consultas por segundo (QPS)?

O único limite é o seu hardware. Em um servidor moderno com processador adequado e interfaces 10GbE, o motor Unbound do Sentinel atinge confortavelmente mais de 100.000 QPS (Queries per Second).
